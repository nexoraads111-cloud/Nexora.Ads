# 📌 Система отзывов без Firebase

Теперь отзывы работают через GitHub Issues и бесплатный GitHub Actions + Telegram.

## Что сделано

- Убрали Firebase из отзывов
- Отзывы загружаются на сайт из GitHub Issues с меткой `approved`
- Форма оставляет новый GitHub Issue с заполненными полями
- GitHub Action будет уведомлять админа в Telegram о новом отзыве

## Как использовать

### 1. Отправка отзыва на сайте

Пользователь нажимает кнопку "Оставить отзыв", заполняет форму и отправляет.
После этого откроется страница создания нового issue в GitHub.
Нажми там **Submit new issue** — это бесплатный способ сохранить отзыв.

### 2. Админ получает уведомление в Telegram

GitHub Action отправляет сообщение в Telegram при каждом новом issue.
Для работы нужно добавить секреты в репозитории:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

### 3. Одобрение отзыва

Чтобы отзыв появился на сайте, админ должен пометить issue меткой `approved`.
Только такие отзывы будут показаны.

## Настройка GitHub Actions

Файл Actions уже добавлен: `.github/workflows/telegram-issue-notify.yml`

1. Перейди в GitHub репозиторий.
2. Открой `Settings` → `Secrets and variables` → `Actions`.
3. Добавь два секрета:
   - `TELEGRAM_BOT_TOKEN` — токен бота (начинается с `862805...`)
   - `TELEGRAM_CHAT_ID` — твой id `6057196483`

## Как настроить метки

1. Открой раздел `Issues` в репозитории.
2. Создай метку `approved`.
3. По желанию создай метку `review` для новых заявок.

## Формат issue

GitHub issue создаётся автоматически с такими полями:

- Name
- Type
- Rating
- Text

Пример:

```
Name: Иван
Type: Создание сайта
Rating: 5
Text:
Отличный опыт, спасибо!
```

## Проверка работы

- На сайте отображаются только issue с меткой `approved`
- Новые отзывы открывают GitHub issue
- GitHub Action отправляет уведомление в Telegram

## Примечание

Это бесплатная система без Firebase: репозиторий служит хранилищем отзывов, а GitHub Actions — уведомляет Telegram.
