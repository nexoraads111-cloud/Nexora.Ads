# 📌 Система отзывов без Firebase и без GitHub Issues

Теперь отзывы работают через Telegram и статический файл `reviews.json`.

## Что сделано

- Убрали Firebase и GitHub Issues из отзыва
- Отзывы на сайте загружаются из файла `reviews.json`
- Форма открывает Telegram для отправки отзыва менеджеру
- Админ добавляет одобренные отзывы вручную в `reviews.json`

## Как использовать

### 1. Отправка отзыва на сайте

Пользователь нажимает кнопку "Оставить отзыв", заполняет форму и нажимает отправить.
После этого откроется Telegram с готовым сообщением менеджеру.

### 2. Админ получает отзыв в Telegram

Пользователь отправляет сообщение менеджеру напрямую через Telegram.
Это полностью бесплатная схема без серверов и без внешних баз данных.

### 3. Одобрение отзыва

Чтобы отзыв появился на сайте, админ добавляет его в файл `reviews.json`.

Пример записи:

```json
{
  "id": 1,
  "name": "Иван",
  "title": "Отличный сайт",
  "type": "Создание сайта",
  "rating": 5,
  "text": "Сайт сделали быстро и красиво.",
  "createdAt": 1700000000000
}
```

### Структура файла

- `id` — уникальный номер отзыва
- `name` — имя клиента
- `title` — заголовок отзыва
- `type` — тип работы
- `rating` — оценка от 1 до 5
- `text` — текст отзыва
- `createdAt` — время публикации в timestamp

## Проверка работы

- На сайте отзывы пришли из `reviews.json`
- Отправка отзыва открывает Telegram
- Система работает без GitHub Issues и без Firebase

## Примечание

Это самый простой бесплатный вариант: форма отправляет отзыв менеджеру в Telegram, а сайт показывает только отзывы, которые админ добавил в `reviews.json`.

## Локальное тестирование и обновление

- Чтобы быстро протестировать сайт локально, запусти в корне проекта простой сервер (Python или `npx serve`) и открой `http://localhost:8000`.
- Проверь, что отзывы загружаются из `reviews.json` и что кнопка отправки отзыва открывает Telegram с заполненным текстом.
- Когда админ одобряет отзыв, добавь запись в `reviews.json` и закоммить изменения в репозиторий:

```bash
git add reviews.json
git commit -m "Add approved review"
git push
```

После пуша на GitHub Pages (если сайт там хостится) изменения станут видны на сайте.
